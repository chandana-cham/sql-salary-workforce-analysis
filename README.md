# Salary & Workforce Strategy Analysis using SQL

## Business Problem
A company wants to design a competitive and cost-effective hiring strategy for data-related roles. This project analyzes global salary data using SQL to support compensation planning, role prioritization, and location-based hiring decisions.

## Objectives
- Analyze salary trends across data roles
- Compare salaries by experience level
- Study employment type distribution
- Understand salary differences by company size
- Identify high-paying and cost-effective locations
- Analyze salary trends over time

## Dataset
Source: Public Data Science Job Salaries dataset  
Each record represents a job role with salary, experience, employment type, company size, and location details.

## SQL Concepts Used
- SELECT, WHERE
- COUNT, DISTINCT
- GROUP BY, HAVING
- Aggregate functions (AVG)
- CASE WHEN
- Window functions (RANK)
- Subqueries

## Key Analysis Performed
- Overall salary benchmarking
- Experience-level salary comparison
- Job-role-wise salary analysis
- Employment type distribution
- Company size vs salary analysis
- Country-wise salary comparison
- Year-over-year salary trend analysis

## Business Insights
- Experience level is the strongest salary driver
- Specialized roles command higher pay
- Full-time roles dominate the market
- Large companies generally pay higher salaries
- Location significantly impacts salary levels
- Salaries have increased over time

## Outcome
This analysis supports data-driven hiring decisions, salary planning, and workforce strategy optimization.
